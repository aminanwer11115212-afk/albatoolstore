# خطة: حذف ذكي متتابع + تعديل كامل من الهيكل الجغرافي

## الهدف

عند حذف اتجاه/ولاية/مدينة/محلية/مجموعة/ترحيل/وجهة:

- **بدون ارتباطات** → رسالة تأكيد بسيطة واحدة، ثم حذف مباشر (السلوك الحالي).
- **مع عملاء مرتبطين أو عناصر فرعية** → نافذة تحذير ذكية تعرض العدّ الفعلي وتقدّم خيارين:
  1. **حذف فقط + فكّ الربط**: تُحذف "دمنهور"، ويصبح `state_id = NULL` لعملائها وتُحذف مدنها/محلياتها التابعة (مع فكّ ربطها أيضاً).
  2. **حذف كامل مع المرتبطين**: تُحذف "دمنهور" وكل مدنها ومحلياتها **و كل العملاء المرتبطين بها**.
  3. إلغاء.

نفس الحوار يُستخدم من:
- قوائم `InlineSearchSelect` داخل حوار العميل.
- الهيكل الجغرافي (بعد إضافة أزرار الإدارة).

بالإضافة إلى ذلك: **الهيكل الجغرافي** يصبح شاشة إدارة كاملة (إضافة/تعديل/حذف على كل مستوى).

## الملفات الجديدة والمعدّلة

### 1) جديد — `src/components/shared/DeleteGeoDialog.tsx`

حوار موحّد يستقبل:
```
{
  entityLabel: "الولاية",      // للعرض
  entityName:  "دمنهور",
  counts: { customers: 12, children: 3, childrenLabel: "مدن" },
  onDeleteOnly: () => Promise<void>,  // فكّ الربط
  onDeleteCascade: () => Promise<void>, // حذف كامل
}
```
- يعرض جدول عدّاد واضح (X عميل مرتبط، Y مدينة تابعة).
- زران: «حذف الولاية فقط (فكّ الربط)» و «حذف الولاية + كل التوابع والعملاء» — الثاني بلون `destructive`.
- إذا كان العدّ = 0 لا يُفتح أصلاً (نستخدم `window.confirm` البسيط الحالي).

### 2) جديد — `src/utils/geoDeleteHelpers.ts`

دوال SQL-side آمنة:
- `getGeoImpact(kind, id)` → يعدّ العملاء المرتبطين مباشرة + المرتبطين عبر الأبناء.
- `deleteGeoOnly(kind, id)` → `UPDATE customers SET <fk>=null WHERE <fk>=id`، ثم حذف الأبناء بنفس المنطق، ثم حذف السجل نفسه.
- `deleteGeoCascade(kind, id)` → يحذف العملاء المرتبطين أوّلاً (بشرط أن يكون ID الوحيد المرتبط بهم — لا نحذف عميل مرتبط باتجاه آخر)، ثم الأبناء، ثم السجل.

لكل من: `region | state | city | locality`. للمجموعة/الترحيل/الوجهة: تحقّق فقط من العملاء المرتبطين (لا أبناء) واستخدم نفس الحوار المبسّط.

### 3) تحديث — `src/components/CustomerFormDialog.tsx`

استبدال دوال `removeRegion/State/City/Locality/...` بـ:
1. حساب `impact = await getGeoImpact(...)`.
2. إن كان `impact.total === 0` → `window.confirm` الحالي ثم حذف مباشر.
3. وإلا → فتح `DeleteGeoDialog` مع الخيارين.
4. بعد النجاح: تحديث القوائم المحلية + `queryClient.invalidateQueries(["customers"])`.

### 4) تحديث — `src/components/customers/GeoStructurePanel.tsx`

إضافة أزرار على كل عقدة (تظهر عند hover):
- **➕** إضافة (ابن): بجانب كل اتجاه/ولاية/مدينة — يفتح prompt سريع (كما في `InlineSearchSelect`).
- **✎** تعديل الاسم: prompt.
- **✕** حذف: يمرّ بنفس منطق `DeleteGeoDialog`.

زر «➕ إضافة اتجاه» أعلى الشجرة.

كل العمليات تستخدم نفس helpers من `geoDeleteHelpers.ts` + دوال إضافة/تعديل من `CustomerFormDialog` (نستخرجها إلى `src/utils/geoMutations.ts` مشترك).

### 5) جديد — `src/utils/geoMutations.ts`

`addRegion / addState / addCity / addLocality / renameGeo(kind,id,name)` — استخراج ما هو مكرّر بين `CustomerFormDialog` و `GeoStructurePanel` في مكان واحد يعتمد على `supabase` مباشرة ويُصدر toast عربية.

### 6) تحديث — `src/pages/CustomersPage.tsx`

- تمرير callback `onDataChanged` للـ `GeoStructurePanel` لإعادة جلب `regions/states/cities/localities` بعد أي تعديل من داخله.
- الاستماع لحدث `geo:changed` (dispatched من helpers) لإعادة الجلب.

## سلوك الحذف — أمثلة

| السيناريو | ما يحدث |
|---|---|
| حذف اتجاه فارغ | تأكيد بسيط → حذف. |
| حذف ولاية «دمنهور» بها 12 عميل و3 مدن | يفتح الحوار الذكي: «12 عميل، 3 مدن مرتبطة. حذف فقط؟ أم حذف مع كل التوابع؟» |
| حذف مدينة بدون عملاء ولا محليات | تأكيد بسيط → حذف. |
| حذف مجموعة عملاء «تجزئة» بها 40 عميل | حوار بسيط: «40 عميل مرتبط، سيتم فكّ ربطهم فقط — تأكيد؟» (لا خيار cascade هنا). |

## أثر جانبي مهم

- «حذف فقط» لا يفقد العملاء — فقط تصبح خانتهم الجغرافية فارغة.
- «حذف كامل» يُشترط أنه لا يحذف عميلاً لديه فواتير/معاملات — إن وُجدت، يُرفض الحذف مع رسالة واضحة تسرد أسماءهم (نستخدم فحصاً على `invoices` و `transactions` قبل التنفيذ).

## اختبارات القبول

1. من حوار العميل → افتح قائمة الولاية → اضغط ✕ على ولاية فارغة → confirm واحد → تختفي.
2. اضغط ✕ على «دمنهور» (بها عملاء) → يفتح `DeleteGeoDialog` مع العدّ الصحيح → «حذف فقط» → تختفي الولاية ويظل العملاء بدون ولاية.
3. نفس السيناريو مع «حذف كامل» → تختفي الولاية والمدن والعملاء (إن لم يكن لهم فواتير).
4. من الهيكل الجغرافي → ➕ على اتجاه → prompt → تُضاف ولاية جديدة وتظهر فوراً.
5. ✎ على مدينة → تعديل الاسم يظهر فوراً وفي حوار العميل التالي.
6. ✕ على مدينة بها محليات + عملاء → نفس الحوار الذكي يعمل من هنا أيضاً.

## خارج النطاق

- لا نلمس المخازن أو المنتجات — الحذف الذكي مقتصر على الجغرافيا/المجموعات/الترحيلات/الوجهات.
- لا نضيف undo — الحذف نهائي بعد التأكيد.
