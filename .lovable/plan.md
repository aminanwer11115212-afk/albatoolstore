# خطة: إضافة الجغرافيا السودانية الكاملة للعملاء

## السياق
الجداول الحالية في "قاعدة البيانات":
- `regions` (الاتجاهات) — حالياً فيها "السودان" و"الشمالي" فقط
- `states` (الولايات) — فيها "الخرطوم" فقط
- `cities` (المدن) — فارغة تقريباً
- `localities` (المحليات) — فارغة تقريباً

الهرمية: **اتجاه → ولاية → مدينة → محلية**، ومكوّن `GeoStructurePanel` يعرضها كشجرة قابلة للبحث.

---

## الخطوة 1 — الاتجاهات الأربعة (regions)
إدراج 4 اتجاهات أساسية إن لم تكن موجودة:
- **الشمال**
- **الجنوب**
- **الشرق**
- **الغرب**

(الاحتفاظ بـ"السودان" و"الشمالي" القديمتين لتجنّب كسر الربط الحالي لـ"الخرطوم" — أو نقل ربطها للاتجاه الجديد في الخطوة 2.)

## الخطوة 2 — 18 ولاية سودانية موزّعة على الاتجاهات

| الاتجاه | الولايات |
|---|---|
| الشمال | الشمالية، نهر النيل، الخرطوم |
| الشرق | البحر الأحمر، كسلا، القضارف |
| الجنوب | الجزيرة، سنار، النيل الأبيض، النيل الأزرق |
| الغرب | شمال كردفان، جنوب كردفان، غرب كردفان، شمال دارفور، جنوب دارفور، شرق دارفور، غرب دارفور، وسط دارفور |

السلوك: لكل ولاية، إذا كانت موجودة بنفس الاسم → تحديث `region_id` لربطها بالاتجاه الصحيح، وإلا إدراجها جديدة.

## الخطوة 3 — المدن الرئيسية لكل ولاية (cities)
العاصمة + أهم 2-4 مدن لكل ولاية، مثال:
- الخرطوم → الخرطوم، أم درمان، بحري
- الشمالية → دنقلا، مروي، كريمة، حلفا
- نهر النيل → الدامر، عطبرة، شندي، بربر
- البحر الأحمر → بورتسودان، سواكن، طوكر، سنكات
- كسلا → كسلا، خشم القربة، حلفا الجديدة
- القضارف → القضارف، الفاو، الحواتة
- الجزيرة → ود مدني، الحصاحيصا، المناقل، الكاملين
- سنار → سنجة، سنار، الدندر، السوكي
- النيل الأبيض → ربك، كوستي، الدويم، تندلتي
- النيل الأزرق → الدمازين، الروصيرص، قيسان، باو
- شمال كردفان → الأبيض، أم روابة، بارا
- جنوب كردفان → كادقلي، الدلنج، رشاد، تلودي
- غرب كردفان → الفولة، بابنوسة، النهود، المجلد
- شمال دارفور → الفاشر، كتم، مليط، الكومة
- جنوب دارفور → نيالا، كاس، عد الفرسان
- شرق دارفور → الضعين، عديلة، شعيرية
- غرب دارفور → الجنينة، فوربرنقا، كلبس، هبيلا
- وسط دارفور → زالنجي، نيرتتي، أزوم، روكيرو

≈ **65 مدينة** إجمالاً.

## الخطوة 4 — المحليات الرئيسية لكل مدينة (localities)
أبرز 1-3 محليات تابعة للمدن الكبرى (الخرطوم/أم درمان/بحري بشكل أوسع، والمدن الإقليمية الأخرى بـ1-2 محلية معروفة)، مثلاً:
- الخرطوم → الخرطوم وسط، جبل أولياء
- أم درمان → كرري، أمبدة، أم بدة
- بحري → شرق النيل، الخرطوم بحري
- بورتسودان → بورتسودان، حلايب
- ود مدني → ود مدني الكبرى، جنوب الجزيرة
- كوستي → كوستي، القطينة
- نيالا → نيالا شمال، نيالا جنوب
- ... (نحو 40 محلية مختارة)

## الخطوة 5 — التنفيذ التقني
- استخدام أداة **إضافة/تعديل البيانات** (`supabase--insert`) لإدراج الصفوف (هذه بيانات، لا تغيير schema).
- النمط لكل مستوى: 
  1. `INSERT … WHERE NOT EXISTS (SELECT 1 … WHERE name = …)` لتجنّب التكرار
  2. `UPDATE … SET parent_id = …` لتصحيح ربط الصفوف الموجودة سلفاً
- استعلام واحد متعدد العبارات (transaction واحد) لضمان سلامة الربط.

## الخطوة 6 — التحقق
بعد التنفيذ:
- عدّاد العنوان في `GeoStructurePanel`: 4 اتجاهات / ~18 ولاية / ~65 مدينة / ~40 محلية
- شجرة العملاء تظهر الاتجاهات الأربعة في الأعلى، يفتح كل اتجاه على ولاياته، وكل ولاية على مدنها، وكل مدينة على محلياتها.
- العملاء الموجودون مسبقاً (`customer.state_id = الخرطوم`) يبقون مرتبطين تلقائياً بالولاية، وبالتالي بالاتجاه الجديد "الشمال".

## ملاحظات
- لن يتم حذف أي بيانات موجودة — فقط إدراج وتحديث ربط.
- إذا أردت توزيعاً مختلفاً للولايات على الاتجاهات (مثلاً اعتبار الخرطوم/الجزيرة/سنار/النيل الأبيض ضمن "الوسط" بدل توزيعها على شمال/جنوب)، أخبرني قبل التنفيذ وأعدّل التوزيع.
