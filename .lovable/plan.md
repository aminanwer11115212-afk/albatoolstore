# خطة التعديلات على صفحة المنتجات وتقرير PDF

## 1) نقل زرَّي «معاينة» و«طباعة/PDF»

- إزالتهما من شريط تقرير الأسعار.
- إضافتهما في شريط أدوات صفحة **إدارة جميع المنتجات** (`isAllProducts`) بجانب زر التصدير/التجميد الحالي، مع تعطيلهما أثناء التصدير.

## 2) إعادة تصميم مخرج PDF ليكون جدولاً على نمط الفواتير

الشكل الجديد يستبدل شبكة البطاقات الصغيرة بجدول Excel-like يشبه قالب الفاتورة:

- **ترويسة الشركة** أعلى كل صفحة مطبوعة تلقائياً (repeating table header):
  اسم الشركة، الشعار إن وُجد، العنوان، الهاتف — تُقرأ من `company_settings` بنفس الحقول المستخدَمة في قالب الفاتورة.
- **عنوان التقرير**: «كتالوج المنتجات» + التاريخ + عدد الأصناف + ملخّص الفلاتر المطبَّقة (مثل «الماركة: Suzuki»).
- **أعمدة الجدول**: #، الصورة (مربعة صغيرة ~35×35mm مع صورة SVG بديلة عند الغياب)، **اسم المنتج** (عمود عريض ظاهر بوضوح)، الفئة، الماركة، المستودع، SKU.
- بدون أعمدة أسعار مطلقاً.
- كل صف `page-break-inside: avoid` + `<thead>` مع `display: table-header-group` لتكرار الترويسة على كل صفحة PDF.
- تذييل الصفحة: «الصفحة X من Y» عبر CSS `@page` counters.

## 3) نافذة المعاينة التفاعلية (Preview Modal)

قبل فتح نافذة الطباعة، تظهر نافذة داخل التطبيق فيها:

- **بحث نصّي** + **فلاتر منسدلة** للفئة، الماركة، المستودع (مبدئياً تُملأ من فلاتر الصفحة الحالية).
- **ترتيب**: حسب الاسم / الفئة / الماركة، تصاعدي أو تنازلي.
- عدّاد مباشر للنتائج المطابقة.
- زران: «طباعة الآن» و«معاينة فقط».
- مثال الاستخدام: اختيار «Suzuki» في الماركة → معاينة → طباعة.

منطق البيانات: قائمة `filtered` الحالية تُمرَّر للـ modal، ثم تُطبَّق طبقة فلترة/ترتيب إضافية داخله قبل توليد HTML.

## 4) إصلاح سلوك مسطرة (Space) في التحديد

- الحذف بالمسطرة يبقى **فقط عند التركيز على الصف نفسه** (خارج خانة الشيك).
- عند التركيز على **خانة الشيك**: المسطرة تُبدّل التحديد فقط (Toggle check) ولا تحذف — حتى الضغطة المزدوجة.
- التنقل بالأسهم يمين/يسار يبقى للتعديل العادي داخل الخلايا.
- الطريقة التقنية: إضافة `data-nospace-delete` على خانة الشيك و`onKeyDown` يستهلك Space ويستدعي `toggleSelected`، مع تعديل `useSpaceToDelete` ليتجاهل الأهداف الحاملة لهذه السمة.

## التفاصيل التقنية

- **الملفات المتأثرة**: `src/pages/ProductsPage.tsx` فقط (لا حاجة لملفات جديدة الآن — نافذة المعاينة تُبنى كمكوّن داخلي محلي).
- **الاستعلامات**: قراءة إضافية من `company_settings` للحقول `company_name, logo_url, address, phone` (استعلام موجود أصلاً في `exportFilteredPdf`، يُوسَّع فقط).
- **CSS للطباعة**:
  ```text
  @page { size: A4; margin: 15mm 10mm 15mm 10mm; }
  thead { display: table-header-group; }
  tr    { page-break-inside: avoid; }
  ```
- **الصورة البديلة**: نفس SVG المضمَّن الحالي (data URI) — يظهر مربّعاً بحجم موحّد داخل الخلية.
- **useSpaceToDelete**: يُضاف فحص `target.closest('[data-nospace-delete]')` لتخطّي الحذف على خانة الشيك.

## معايير القبول

1. زرّا الطباعة والمعاينة يظهران فقط في صفحة إدارة كل المنتجات.
2. PDF فيه ترويسة شركة + جدول بأعمدة (صورة/اسم/فئة/ماركة/مستودع/SKU) بدون أسعار، والترويسة تتكرّر على كل صفحة.
3. المعاينة تسمح باختيار ماركة Suzuki (مثلاً) وترتيب حسب الاسم تنازلياً ثم الطباعة.
4. الضغط على المسطرة داخل خانة الشيك يُبدّل التحديد فقط ولا يحذف.
5. الحذف بالمسطرة المزدوجة يعمل فقط عند التركيز على الصف (لا الخانة).
