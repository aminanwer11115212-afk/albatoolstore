## الهدف العام

توحيد عناصر الجوال مع الديسكتوب في صفحات: عرض السعر، الفاتورة، أمر الشراء، المرتجع — مع تخطيط جوال أنسب (بطاقات بدل جدول الأصناف، رأس عميل مرتّب، شريط ملخّص ثابت أسفل). **بدون أي تغيير في منطق الحفظ، الأتمتة، أو قاعدة البيانات** — نفس الدوال (`updateRow / pickProductIntoRow / removeRow / addQuickRowToTable / totals`) تعمل من الواجهتين.

## التقسيم إلى دفعات

| الدفعة | الصفحات | النطاق |
|---|---|---|
| **1 (هذه)** | `QuoteCreatePage.tsx` | إنشاء/تعديل عرض السعر (الصفحة الظاهرة في اللقطة) |
| 2 | `InvoiceCreatePage.tsx` | إنشاء/تعديل فاتورة (يستفيد من نفس النمط) |
| 3 | `PurchaseCreatePage.tsx` + `StockReturnCreatePage.tsx` | الشراء + المرتجع |
| 4 | `InvoicesPage` / `QuotesPage` / `PurchasePage` / `StockReturnPage` | صفحات القوائم (مزيج تلقائي: بطاقات ≤640px وجدول ≥641px) |
| 5 | `InvoiceViewPage` / `QuoteViewPage` / `StockReturnViewPage` | لمسات على صفحات العرض إن لزم |

كل دفعة تأتي بعد قبول الدفعة السابقة، حتى لو ظهر اختلاف بسيط نعالجه قبل الانتقال.

## الدفعة 1 — QuoteCreatePage (مفصّلة)

### المشاكل في اللقطة الحالية
1. رأس البطاقة البرتقالي: حقل «اسم العميل» يطغى على تسمية «رقم العرض (مقترح)» وتتداخل النصوص.
2. صف «الإضافة السريعة»: 6 عناصر بشبكة `quickGrid` ديسكتوبية → على الجوال يتكدّس ويختفي بعض الحقول.
3. جدول الأصناف (`excel-table`) بـ 8 أعمدة + handles لتغيير عرض الأعمدة + suggestions portal → غير قابل للاستخدام على شاشة ~390px (الجدول ظاهر فارغاً بحجم كبير بدون تمرير أفقي مع الكسر لأعمدة 30px).
4. شريط الملخّص السفلي (FreePositionToolbar) يستخدم drag & drop وSummaryChip بحجم ديسكتوب.

### الحل

1) **حقن `useIsMobile()`** في أعلى المكوّن:
   ```tsx
   import { useIsMobile } from "@/hooks/use-mobile";
   const isMobile = useIsMobile();
   ```

2) **رأس بطاقة العميل**: عند `isMobile` تبديل ترتيب الأعمدة إلى عمود واحد بـ `flex-direction: column` وزيادة `gap`؛ يضمن أن كل حقل بسطر مستقل بتسميته. لا تغيير على المحتوى أو الحالة.

3) **صف الإضافة السريعة**: عند `isMobile`، استبدال `gridTemplateColumns: quickGrid(...)` بـ `grid-template-columns: 1fr 1fr` لحقول رقمية (الكمية + المحلي + الأجنبي + معدل التحويل) + سطر مستقل كامل العرض لـ«ابحث عن منتج» وسطر آخر لزر «+ إضافة» بعرض كامل و `min-height: 44px`.

4) **جدول الأصناف → بطاقات على الجوال** (الأهم):
   - عند `isMobile` لا نعرض `<table className="excel-table">` بل نعرض `<MobileItemCardList>` يلفّ نفس `visibleRows` ويُنشئ بطاقة لكل بند:
     ```
     ┌───────────────────────────────────────────┐
     │ ☑  📝 #1                              ✕  │  ← شريط علوي: تحديد، رقم، حذف
     ├───────────────────────────────────────────┤
     │ [بحث عن منتج: input بعرض كامل]            │
     │ ┌─ الكمية ─┐ ┌─ المحلي ─┐                  │
     │ │   2     │ │  4,500   │                  │
     │ └─────────┘ └─────────┘                  │
     │ ┌ الأجنبي $┐ ┌ معدل التحويل ┐             │
     │ │   15.0   │ │   600       │             │
     │ └─────────┘ └─────────────┘              │
     │ الإجمالي: 9,000 SDG  [📝 ملاحظة]          │
     └───────────────────────────────────────────┘
     ```
   - كل input في البطاقة يستخدم **نفس** `data-nav-table="quote-items"` و`onChange`/`onKeyDown` الموجودة في صف الجدول، حتى يبقى pickProductIntoRow / SuggestionsPortal / Space-to-delete / handleNav يعملون بدون تغيير.
   - الملاحظة تظهر داخل البطاقة في سطر أسفل بدل صف منفصل.

5) **شريط الملخّص السفلي**: عند `isMobile` نُغلق وضع التخصيص ونعرض شريطاً ثابتاً (`sticky bottom-0`) من 3 عناصر فقط: **الخصم العام، المجموع، زر الحفظ** — مع زر «المزيد» يفتح Sheet/Drawer (`@/components/ui/sheet`) فيه باقي الأزرار (تحويل لفاتورة، تغيير الحالة، إرسال واتساب، مسح…). الديسكتوب يبقى كما هو حرفياً.

6) **عداد البنود + خصم BulkActions**: على الجوال يصبح شريطاً علوياً ثابتاً (sticky top-0) داخل قسم الأصناف.

### قواعد عامة
- لا تغيير على `index.css` العام.
- استخدام رموز تصميمية فقط (Tailwind tokens) — لا ألوان مباشرة في المكوّنات الجديدة.
- كل input بحجم خط ≥ 16px (للالتزام بالقاعدة الحالية في `index.css`).
- ارتفاع لمس أدنى 44px لكل زر/حقل على الجوال.
- لا حذف لأي زر/خاصية من الديسكتوب — كل شيء يبقى متوفراً عبر «المزيد» أو داخل البطاقة.

### الملفات المعدّلة في الدفعة 1
- `src/pages/QuoteCreatePage.tsx` فقط (تعديل موضعي + مكوّن داخلي `MobileItemCard`).

### التحقق
- فتح `/quotes/new` على عرض جوال 390px: لا تداخل نصوص، الإدخال سلس بالأصابع، البحث عن المنتجات والاقتراحات تعمل، إضافة/تعديل/حذف بند يعمل، الإجمالي يتحدّث، الحفظ والتحويل لفاتورة يعملان.
- فتح نفس الصفحة على ديسكتوب 1280px: لا تغيير مرئي إطلاقاً (الجدول والشريط القديمان كما هما).

أنتقل للدفعة 2 (`InvoiceCreatePage`) باستخدام نفس نمط `MobileItemCard` المعمّم.
