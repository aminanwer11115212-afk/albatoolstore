## الهدف
إضافة زر "إعادة القص" على الصور بعد حفظها في:
1. صورة المنتج (نموذج المنتج + الصورة الصغيرة داخل جدول المنتجات)
2. مرفقات الصور في الفواتير / عروض الأسعار / المشتريات
3. شعار الشركة في إعدادات الشركة

بعد الاعتماد يُستبدل الملف في التخزين ويُحدَّث سجل قاعدة البيانات مع الحفاظ على نفس المعرّف.

## آلية العمل

```text
[صورة محفوظة] → زر ✂️ → تنزيل URL كـ Blob → File → ImageCropDialog (بمعاينة) → قص واعتماد
                                                                                     ↓
                                             رفع الملف الجديد لنفس المسار (upsert) + تحديث DB → توست نجاح
```

## التفاصيل التقنية

### 1) أداة مشتركة `src/utils/fetchImageAsFile.ts`
- `fetchImageAsFile(url: string, fallbackName?: string): Promise<File>` — تنزّل الصورة عبر `fetch`، وتستخرج mime من `Content-Type`، ثم تُنشئ `File`. تدعم روابط `signed URL` والعامة.

### 2) هوك مشترك `src/hooks/useRecropImage.ts`
- يفتح `ImageCropDialog` على صورة موجودة
- يستقبل callback `onCropped(newFile)` ليتولّى الاستدعاء الرفع/التحديث الخاص به
- يُظهر توست تحميل/نجاح/خطأ

### 3) صورة المنتج — `src/pages/ProductsPage.tsx` و `QuickAddProductDialog.tsx`
- زر صغير (أيقونة مقص) فوق الصورة الحالية داخل النموذج
- زر مقابل داخل الجدول بجانب الصورة المصغّرة
- المسار: نفس مسار الصورة السابق في `product-images` مع `upsert: true` ليبقى الـ URL ثابتًا (يُضاف `?v=timestamp` لكسر الكاش)، ثم `update products set image_url` إذا اختلف.

### 4) مرفقات الوثائق — الثلاث حوارات
داخل عناصر القائمة عند كون `file_type` صورة:
- زر "قص" بجوار "تنزيل" و"حذف"
- يستخدم نفس آلية `upsert` على مسار المرفق
- يحدّث سجل `*_attachments` بحقلَي `file_size` و`file_type` والوقت `updated_at`

### 5) شعار الشركة — `src/pages/CompanySettingsPage.tsx`
- زر صغير على معاينة الشعار الحالي يعيد فتح حوار القصّ ويرفع بنفس المسار

### 6) اختبارات جديدة `src/test/recropFlow.test.ts`
- التحقّق أن `fetchImageAsFile` يستنتج mime الصحيح (mock لـ `fetch`)
- التحقّق أن `useRecropImage` يفتح الحوار ويستدعي callback بالملف المقصوص
- التحقّق أن اسم الملف الناتج يحمل لاحقة `-recropped`

## نقاط سلوكية
- زر إعادة القص يظهر فقط للصور (`image/*`)، ليس لـ PDF أو مستندات Office
- بعد النجاح يُعاد تحميل قائمة المرفقات / بيانات المنتج تلقائيًا
- الحفاظ على نفس مسار التخزين يُبقي أي روابط سابقة (طباعة/PDF) صالحة
- إذا فشل التنزيل (رابط منتهي) → رسالة واضحة "تعذّر تحميل الصورة لإعادة القص"

## نطاق مستبعد
- لا نغيّر منطق الحذف/الاستعادة أو صلاحيات RLS
- لا نضيف نسخة تاريخية (versions) للصور — الاستبدال فوري
