# تحويل قسم التغليف في كشف الترحيلات إلى أسطر نصية

ملف واحد: `src/components/dispatch/DispatchPrintPreview.tsx` — تعديل `renderPackagingHtml` فقط.

## الفكرة
بدل الجدول الحالي، اعرض كل صنف تغليف كسطر نصي مرقّم بنفس ترتيب وصياغة تقرير تغليف الفاتورة (`generatePackagingReportHTML`):

```
1) 2 — كرتونة صغيرة استارتر 200 11 سنة اصلي — × 5
2) 1 — كرتون كبير — اسم الصنف — × 12
```

التركيب لكل سطر:
- الرقم التسلسلي (1، 2، ...)
- عدد الطرود `packs_count` (افتراضي 1)
- شرطة فاصلة
- اسم نوع التغليف بخط أسمك (من `packaging_types.name` للسجل الأب) ثم اسم المنتج
- إن وُجد `pieces_per_pack > 1`: ` — × {pieces}`

## التنفيذ
- داخل `renderPackagingHtml`، إنشاء قائمة `<ol class="d-pk-lines">` بدل `<table>`.
- تمرير اسم النوع لكل صنف عبر `_typeName` المخزّن مسبقاً على `r.packaging_types?.name` (يُمرَّر مع كل عنصر عند جمع `allItems`).
- لو الفاتورة لا تحوي `items` تفصيلية، استخدم كل سجل تغليف نفسه كسطر واحد (`packs_count` + `packaging_types.name`).
- الحفاظ على شرائح `typesBits` الموجودة فوق القائمة (اختياري — يمكن إلغاؤها لأنها صارت مكررة) → سنُلغيها لتجنّب التكرار.
- إزالة `<table>` و CSS الخاص بـ `.d-pk-table` و `.d-pk-chips`، وإضافة CSS بسيط لـ `.d-pk-lines` و `.d-pk-line` (سطور 11px، تباعد 2px، نقاط ترقيم بنمط عربي عادي).

## ما لا يتغيّر
- لا تعديل في قاعدة البيانات.
- لا تغيير على بقية البطاقة (الزبون/الناقل/الوجهة/الملاحظات).
- إجماليات الطرود/الوزن تبقى مخفية كما هي الآن.
- شريط الملخص العلوي (التاريخ + عدد الفواتير) كما هو.
