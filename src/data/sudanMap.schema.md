# توثيق `sudanMap.config.json`

ملف بيانات خريطة السودان. **عدّل هذا الملف فقط** لتغيير الأشكال أو الأسماء
أو إضافة/حذف منطقة — لا حاجة لتعديل أي كود React.

## البنية

```jsonc
{
  "viewBox": "0 0 400 500",   // نظام إحداثيات الـ SVG. لا تغيّره إلا لو أعدت رسم الخريطة كاملة.
  "regions": [
    {
      "id":       "north",                       // slug ثابت — يُستخدم للربط الداخلي فقط
      "name":     "الشمالية",                     // الاسم المعروض ويُطابق regions.name في قاعدة البيانات
      "aliases":  ["شمال", "northern"],           // أسماء بديلة لربط سطر قاعدة البيانات حتى لو اختلفت الكتابة
      "path":     "M120 40 L280 40 ... Z",       // مسار SVG (attribute d) — أي polygon مغلق
      "labelXY":  [200, 105]                     // موقع النص داخل المنطقة (مركز التسمية)
    }
  ]
}
```

## خصائص شكل SVG

| المفتاح | الشرح |
|--------|------|
| `viewBox` | نظام إحداثيات موحّد. الافتراضي `0 0 400 500` (عرض 400 × ارتفاع 500). |
| `path.d`  | يبدأ بـ `M x y` (نقطة البداية) ثم `L x y` لكل ركن، وينتهي بـ `Z` لغلق الشكل. |
| المحاور   | x يزيد لليمين، y يزيد للأسفل (نظام SVG القياسي). |
| `labelXY` | إحداثيات مركز الاسم داخل المنطقة. اضبطها بصرياً بعد تعديل الشكل. |

## إضافة منطقة جديدة

1. ارسم شكل polygon على الشبكة `400 × 500` وسجّل مسار `d`.
2. أضف كائناً في مصفوفة `regions` بـ `id` فريد و`name` مطابق لسطر في جدول `regions`.
3. عدّد `aliases` بأسماء إضافية إن كان الاسم في قاعدة البيانات قد يُكتب بشكل مختلف.
4. احفظ — تظهر المنطقة تلقائياً في `LocationPicker`.

## نصائح

- **ألوان وحدود** لا تُخزّن هنا. مصدرها الوحيد `index.css` (`--primary`, `--muted`, `--border`).
- **لا تكرّر `id`** بين المناطق — يُستخدم كمفتاح React.
- إن أضفت منطقة جديدة لكنها ظهرت بلون رمادي فقط بعد النقر مع رسالة «غير موجودة»،
  تأكّد من وجود سطر مطابق في جدول `regions` (اسم أو slug).
